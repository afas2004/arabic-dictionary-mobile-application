Why this split works for beginners:

The list page never shows more than one Arabic word, one English gloss, and one small sub-line. A user who only reads Arabic without grammar knowledge can scan and tap without being overwhelmed.

All the grammar (POS, conjugation tables, root family, examples) is gated behind the detail page's three tabs. A beginner can safely ignore them on day one; a more advanced learner discovers them when they tap through.

The three tabs map cleanly to the three questions a dictionary user asks in order: "what does this mean?" → "what shape is this word taking?" → "what are its cousins?"

Colour is limited to the tier sub-line only — no coloured card backgrounds, no coloured borders. This keeps the visual noise floor low so the Arabic text itself remains the visual focus.

The rules, restated simply

List page (the ifs, flattened):

Every result card shows exactly three things — the Arabic lemma, an English gloss, and one small coloured line. That coloured line is green if the match is direct (T1) or a conjugation (T2), blue if any clitics were stripped (T3 through T6), amber if the fallback pattern matcher fired (T7). That's it. No badges. No coloured cards. No multi-line explanations.

Homographs get a quiet secondary row, not a popup. If the cascade found more than one strong candidate for the same query, the top result takes the full card; the alternatives appear as compact greyed rows beneath. One-tap access, zero clutter.

Detail page, three tabs:

Tab 1 — Meaning. Definitions, part of speech, one example sentence.

Tab 2 — Forms. Full paradigm for verbs (past, present, imperative, passive); declension grid for nouns (singular, dual, plural, construct). The row the user searched for is highlighted.

Tab 3 — Root. All lexicon entries sharing the three-letter root, each tappable to navigate sideways through the family.

Implementation note: everything above is data the new stemmer already produces. The tier pill's colour is driven by StemResult.source; the clitic line by StemResult.strippedClitics; the root chip and Tab 3 by StemResult.extractedRoot (or the resolved lexicon row's root column for T1 hits). The data pipeline does not change — only the rendering layer does.
